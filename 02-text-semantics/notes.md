# 第二章：文本语义标签

## 核心概念

### 1. 语义（Semantics）是什么？

**语义**指标签所传达的"含义"，而不是"样式"。

这是初学者最容易忽视的点，也是后续写好 React 组件的基础。

| 标签 | 语义含义 | 默认视觉效果 |
|------|----------|--------------|
| `<strong>` | 内容非常重要、紧急 | 粗体 |
| `<b>` | 文字加粗（无语义含义） | 粗体 |
| `<em>` | 内容需要强调（语气强调） | 斜体 |
| `<i>` | 斜体（无语义含义） | 斜体 |

**为什么要用语义标签而不是样式标签？**

1. **可访问性**：屏幕阅读器读到 `<strong>` 时会加重语气，`<b>` 则不会
2. **SEO**：搜索引擎对 `<strong>` 内的词语赋予更高权重
3. **可维护性**：代码的含义一目了然
4. **样式解耦**：即使 CSS 被禁用，内容结构依然有意义

类比：在 TypeScript 中，你可以用 `any` 解决所有类型问题，但使用精确类型会让代码更可靠、更易维护。语义化 HTML 是同样的道理。

### 2. 标题层级

```html
<h1>页面主标题</h1>          <!-- 每页只有一个，通常是网站名或文章标题 -->
<h2>章节标题</h2>
<h3>子章节标题</h3>
<h4>更细分的标题</h4>
<h5>很少用到</h5>
<h6>几乎不用</h6>
```

**重要规则：**
- `<h1>` 每页只能有一个（SEO 和可访问性的要求）
- 标题层级必须按顺序降低，不能跳级（h1 → h3，跳过 h2 是错误的）
- **不要为了"字大一点"而用 h1**——用 CSS/Tailwind 控制大小

**实际开发中的常见错误：**
```html
<!-- 错误：用标题标签控制字号 -->
<h3>这其实是正文，我只是想让它大一点</h3>

<!-- 正确：保持语义，用 CSS 控制样式 -->
<p class="text-xl font-bold">这是正文，通过 Tailwind 控制大小</p>
```

### 3. 段落与内联元素

```html
<!-- 块级元素：独占一行 -->
<p>这是一个段落。段落与段落之间有默认的上下间距。</p>
<p>这是第二个段落。注意不要用连续 <br> 模拟段落间距。</p>

<!-- 内联元素：在行内流动 -->
<p>
  这句话中有<strong>非常重要</strong>的词，
  也有<em>需要强调</em>的部分，
  以及<code>代码片段</code>的样式。
</p>
```

**`<br>` 的正确用法：**
```html
<!-- 正确：真正的换行需求（诗歌、地址） -->
<p>
  北京市朝阳区<br />
  某某路 123 号<br />
  邮编：100000
</p>

<!-- 错误：用 br 制造段落间距（应该用两个 <p>） -->
<p>第一段</p>
<br />
<br />
<p>第二段</p>
```

### 4. 常用文本语义标签

```html
<!-- 代码相关 -->
<code>console.log("hello")</code>    <!-- 行内代码 -->
<pre><code>
function greet() {
  return "hello";
}
</code></pre>                          <!-- 代码块（pre 保留缩进，code 表示代码） -->

<!-- 引用 -->
<blockquote cite="https://example.com">
  这是一段引用文字，来自外部来源。
</blockquote>

<!-- 删除线与下划线（语义用途） -->
<del>原价 199 元</del>               <!-- 表示删除/不再有效的内容 -->
<ins>现价 99 元</ins>                <!-- 表示新插入的内容 -->

<!-- 上标和下标 -->
<p>水的化学式：H<sub>2</sub>O</p>
<p>面积：10<sup>2</sup> = 100</p>

<!-- 缩写 -->
<abbr title="HyperText Markup Language">HTML</abbr>
<!-- 鼠标悬停时显示完整含义 -->

<!-- 时间（机器可读） -->
<time datetime="2026-03-17">2026 年 3 月 17 日</time>
```

### 5. `<div>` 与 `<span>` 的正确定位

`<div>` 和 `<span>` 是**无语义**的容器，仅用于布局或样式分组：

```html
<!-- div：块级无语义容器（当没有合适的语义标签时使用） -->
<div class="card">
  <h2>卡片标题</h2>
  <p>卡片内容</p>
</div>

<!-- span：内联无语义容器（给部分文字加样式时使用） -->
<p>价格：<span class="text-red-500">¥99</span></p>
```

**原则：优先使用语义标签，实在没有合适的才用 div/span。**

## React / Next.js 对照

### 组件中的语义化

```tsx
// 不好：全用 div
function BlogPost() {
  return (
    <div>
      <div className="text-2xl font-bold">文章标题</div>
      <div className="text-gray-500">作者：张三</div>
      <div>文章内容...</div>
    </div>
  );
}

// 好：使用语义标签，配合 Tailwind
function BlogPost() {
  return (
    <article>
      <h1 className="text-2xl font-bold">文章标题</h1>
      <p className="text-gray-500">作者：张三</p>
      <p>文章内容...</p>
    </article>
  );
}
```

第二种写法的优势：
- 搜索引擎更能理解文章结构
- 屏幕阅读器能正确识别文章和标题
- 代码含义更清晰，维护成本更低

### Tailwind 与语义分离

Tailwind 的核心哲学是"样式归样式，结构归结构"：

```tsx
// 结构（HTML 语义）告诉机器"这是什么"
// Tailwind 类名告诉浏览器"这长什么样"
<strong className="text-red-600 font-bold">重要警告</strong>

// 用 CSS 把 strong 改成绿色，不影响它"重要"的语义
// 这种解耦让你可以自由改变设计，而不破坏结构
```

---

## 本章要点总结

1. 语义标签传递含义，样式标签只影响外观——始终优先用语义标签
2. `<h1>` 每页一个，标题层级不能跳级
3. `<strong>` ≠ `<b>`，`<em>` ≠ `<i>`
4. `<br>` 只用于真正需要换行的内容，不用于制造间距
5. `<div>` 和 `<span>` 是最后的选择，不是第一选择
